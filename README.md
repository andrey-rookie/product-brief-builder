# Product Brief Builder

Локальный инструмент для подготовки структурированного брифа по IT-продукту.

## Что умеет

- Собирает бизнес-вводные (проблема, аудитория, KPI, ограничения).
- Добавляет отдельные слои `Роли и права`, `User Journey`, `Состояния/События/Метрики/Экономика/Данные/Edge cases`.
- Поддерживает модульную структуру: `Модуль -> Функционал -> Пользовательские сценарии`.
- Разделяет функционал на `Бизнес-результат` и `Acceptance Criteria`.
- Структурирует сценарии: `Actor`, `Шаги`, `Результат`, `Альтернативный путь`.
- Поддерживает сворачивание на уровнях модуля, функционала и сценария.
- Сохраняет черновик в `localStorage` браузера (`product-brief-draft-v2`).
- Экспортирует бриф в `.md` и `.json`.

## Как использовать

1. Открой файл `/Users/andrei/Desktop/untitled folder/index.html` в браузере.
2. Заполни разделы и добавь нужные модули/функции.
3. Нажми `Скачать .md` и/или `Скачать .json`.
4. Загрузи файл в ChatGPT/Codex и попроси разработать решение на основе брифа.

## Рекомендованный формат ввода

- `Acceptance Criteria`: по одному критерию на строку (инструмент нормализует в маркированный список).
- `Шаги` сценария: по одному шагу на строку (инструмент нормализует в нумерованный список).

## Рекомендация для промпта при загрузке брифа

Пример:

```text
На основе приложенного product brief сделай:
1) архитектуру
2) пошаговый технический план
3) MVP scope
4) backlog по приоритетам
5) риски и вопросы для уточнения
```
